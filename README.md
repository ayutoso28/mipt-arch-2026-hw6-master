# Отчет об исправленных проблемах поддерживаемости

В ходе рефакторинга были выявлены и устранены следующие нарушения принципов проектирования и чистого кода.

## 1. Нарушения SOLID

| Принцип | Выявленная проблема | Исправление |
| :--- | :--- | :--- |
| **ISP** (Interface Segregation) | `CurrencyConverter` требовал реализации 4 методов для каждой валюты, что заставляло классы зависеть от ненужных им методов. | Интерфейс упрощен до одного метода `convert(amount)`. |
| **LSP** (Liskov Substitution) | Методы-заглушки (stubs) возвращали `None` и печатали текст вместо конвертации, нарушая контракт базового класса. | Убраны все методы-заглушки. Каждый класс теперь корректно выполняет только свою задачу через единый интерфейс. |
| **SRP** (Single Responsibility) | Конвертеры совмещали логику API-запросов, обработки ошибок, логирования, кэширования и самой конвертации. | Логика работы с сетью и инфраструктура вынесены в базовый класс. Подклассы отвечают только за определение целевой валюты. |
| **OCP** (Open-Closed) | Для добавления новой валюты требовалось изменять существующий код и копировать большие блоки логики. | Новая валюта добавляется созданием класса из 3-5 строк без изменения логики обхода в `main.py`. |

## 2. DRY (Don't Repeat Yourself)

| Тип дублирования | Описание | Результат исправления |
| :--- | :--- | :--- |
| **Логика API и Retry** | Код запросов к `exchangerate-api.com` и логика повторных попыток были скопированы между файлами. | Логика централизована в базовом классе `CurrencyConverter`. |
| **Константы** | URL API и параметры таймаутов повторялись во всех конвертерах. | Вынесены в атрибуты класса/константы базового класса. |
| **Интерфейс пользователя** | В `main.py` блоки кода для вызова каждого конвертера были идентичны. | Реализован полиморфный цикл по списку конвертеров. |

## 3. KISS и YAGNI

*   **KISS (Keep It Simple, Stupid):** Сложная структура с множеством файлов, содержащих дубликаты, заменена на лаконичную иерархию. Объем кода в конкретных конвертерах сократился в 10 раз.
*   **YAGNI (You Ain't Gonna Need It):**
    *   Удален механизм файлового кэширования в `UsdCnyConverter`, который был избыточен для данной задачи и не использовался в других конвертерах.
    *   Удален неиспользуемый импорт `asyncio` в `main.py`.

## 4. Чистый код (Clean Code)

*   **Устранение "магических строк":** Коды валют и настройки API теперь управляются более явно.
*   **Типизация:** Добавлены Type Hints для аргументов и возвращаемых значений методов.
*   **Обработка ошибок:** Внедрена единая система обработки исключений `requests` и логирования ошибок через стандартную библиотеку `logging`.
*   **Именование:** Методы переименованы в универсальные (вместо `convert_usd_to_eur` теперь просто `convert`), что позволяет использовать их полиморфно.
